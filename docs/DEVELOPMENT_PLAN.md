# 开发计划

## 项目目标

交付一个开源的 OBS Studio 本地实时美颜视频滤镜：支持 Windows x64 与 Apple Silicon macOS（arm64），最低兼容 OBS Studio 31，首版中文界面。

## 里程碑

| 阶段 | 工作内容 | 主要产出 | 验收条件 |
| --- | --- | --- | --- |
| P0：插件与渲染骨架 | OBS Filter 注册、中文 UI、跨端 Shader、打包目录 | 可加载的 `.dll` / `.plugin` 骨架；基础美颜参数 | macOS arm64 可构建、安装包结构正确；Windows 可配置构建。 |
| P1：人脸区域能力 | 集成 MediaPipe Face Landmarker、异步帧处理、脸部轮廓与五官保护 mask | 稳定的局部美颜 mask | 单人移动、转头与短暂遮挡时无明显抖动；渲染线程不等待推理。 |
| P2：体验与性能 | 多人脸、时序平滑、自动降级、预设、性能监控与错误恢复 | 自动/兼容/高质量模式完整可用 | 推荐设备 1080p30 稳定；资源不足时自动降级而不黑屏或阻塞。 |
| P3：跨端交付 | Windows 与 macOS 构建流水线、安装包、签名/公证流程、兼容性测试 | 可发布的双端测试版本 | Windows x64、Apple Silicon macOS 实机安装与 OBS 场景切换验证通过。 |
| P4：高阶扩展（独立评审） | 瘦脸、大眼、妆容、瑕疵修复 | 单独的模型、算法与 UI 方案 | 每项功能通过审美、性能、许可证与稳定性评审后才合并。 |

## 当前进度

- P0：已完成 macOS arm64 构建与安装包结构验证。
- P1：进行中。已固定 MediaPipe 0.10.35、Face Landmarker 模型与许可证台账；已实现关键点转换、多人脸观测关联与时序平滑、只保留最新帧的后台推理工作器、最长边 256px/每秒最多 10 帧的 OBS 双 staging 纹理帧桥，以及按双眼连线旋转对齐、保护眼睛和嘴部的 GPU 脸部椭圆 mask，并完成 Apple Silicon MediaPipe + OpenCV 原生运行时、模型加载、真实推理与工作器测试。已完成 mask 羽化与诊断预览，并在隔离路径下通过 OBS 31.0.2 的插件加载/卸载 smoke test（零内存泄漏）；后续是含滤镜实例的 OBS 场景验证、遮挡处理和边缘调校。
- P2：进行中。已补齐磨皮、提亮、红润与锐化的独立开关，并为既有设置提供默认开启的迁移；迁移单元测试覆盖 P0、v1 与当前 v2 设置。已接入后台推理耗时、自动采样降级与恢复滞回、属性页性能状态，以及最多检测 8 张脸后按面积处理前 4 张的策略。双端负载实测待完成。
- P3：进行中。构建系统已硬性限制 Apple Silicon macOS 与 Windows x64，且已验证 Intel Mac 配置会失败；Windows 原生构建、安装与实机验证待完成。
- P4：未开始。

## P1 实施顺序

1. 固化模型包来源、版本、SHA-256 与 Apache-2.0 许可证记录。（已完成，模型按脚本复现下载。）
2. 建立推理适配层，隔离 MediaPipe、Core ML 与 Windows 后端差异。（已完成 MediaPipe C API 适配层。）
3. 从 OBS 图形纹理安全复制低分辨率帧并提交至后台工作线程。（已完成；双 staging 读回与工作器均已接入。）
4. 使用关键点构建脸部轮廓、眼睛/眉毛/嘴唇保护区 mask。（已完成脸部椭圆、眼睛和嘴部保护；眉毛与像素级皮肤区域留待后续模型评审。）
5. 对 mask 做跟踪、时序平滑、羽化与异常回退。（已完成关键点时序平滑、500ms 轨迹失效、可调羽化、诊断预览、运行状态提示，以及创建或运行期推理失败时的兼容回退；遮挡处理待完善。）
6. 将 mask 上传到 Shader，在完整分辨率画面完成局部磨皮、提亮、红润与锐化。（已完成。）
7. 在单人、多人与不同光照素材上测试效果和耗时。（验收清单已就绪，等待双端实机执行。）

## 质量门槛

- 每次合并前至少完成目标平台编译；涉及 macOS 包时完成 bundle、签名与资源校验。
- 所有第三方库、模型和权重进入仓库前必须更新 `docs/THIRD_PARTY.md`。
- 不在 OBS 渲染线程执行模型推理；推理超时使用最近有效 mask 或降级。
- P1 不承诺瘦脸、妆容或祛痘；这些能力进入 P4 单独评审。

## 已知依赖

- OBS Studio 31+ 开发头文件与 `libobs`。
- MediaPipe Face Landmarker 运行时与经过许可证审查的模型资产。
- Apple Silicon P1 运行时还需要最小 OpenCV 3.4.11 Core/Imgproc 动态库；复现脚本会将其与 MediaPipe 一并构建和打包。
- Windows 构建机（Visual Studio 2022）用于 x64 原生编译与实机测试；运行时脚本使用 MediaPipe 0.10.35 与 OpenCV 3.4.10 Windows x64 包。
- macOS 正式发布阶段需要 Apple Developer ID 签名与 notarization。
