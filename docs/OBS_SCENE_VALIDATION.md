# OBS 场景验收清单

本清单用于 P1/P3 的人工验收。每个目标平台都应使用对应架构的 OBS Studio 31+、新建的本地测试场景和本项目构建出的安装包完成一次。

## 启动与安装

1. 安装插件后启动 OBS，确认日志包含模块加载及 `filter instance created`，且没有动态库、模型或签名加载错误；移除滤镜或退出 OBS 后确认对应的 `filter instance destroyed`。
2. 为摄像头源添加“高级美颜”滤镜，确认属性页为中文，预设、质量模式、羽化和诊断开关均可见。
3. 确认“运行状态”为 `MediaPipe 人脸区域已启用`；故意移走模型后重启，确认显示“模型不可用（已回退为兼容模式）”，画面仍可正常输出。
4. 保存场景集合、完整退出 OBS、重新启动并打开同一集合：滤镜、预设、各开关与数值必须恢复；移除滤镜后再次保存重启，确认它不再出现。
5. 建立至少两个本地测试场景，在两者之间连续切换 20 次；画面与滤镜状态必须正确恢复，期间不得崩溃、黑屏或丢失滤镜。

## 人脸区域与视觉效果

1. 单人正脸、缓慢左右移动、抬头低头各 10 秒：脸部轮廓稳定，眼睛和嘴部不出现明显涂抹。
2. 头部左右倾斜约 30 度各 10 秒：开启“显示人脸区域（诊断）”，确认区域和五官保护区随双眼方向对齐，没有明显偏离。
3. 两人同框、进出画面各 10 秒：两个脸部区域独立，任一人离开后不应残留超过 500ms；可安排 5–8 人同框，确认只处理面积最大的 4 张脸。
4. 手部短暂遮挡半张脸、快速转头、背对镜头各一次：不得闪黑、卡死或让局部 mask 长时间停留在错误位置。
5. 在暖光、冷光、低照度和高对比度条件下各观察 10 秒：只对人脸区域产生效果；关闭滤镜后应恢复原画面。

## 性能、回退与稳定性

1. 分别以 720p30、1080p30、1080p60 本地视频源或摄像头源测试。每个分辨率/帧率组合至少覆盖自动、兼容与高质量模式各 60 秒，并记录 OBS“统计”窗口的渲染延迟、渲染/编码丢帧及插件“性能状态”。
2. 在自动和高质量模式间切换，确认画面不中断；兼容模式必须不依赖模型并保持全画面 Shader 效果。
3. 对 1080p30（推荐机型）或目标发布所声明的最高档位持续运行 2 小时，其中至少包含每 15 分钟一次场景切换。记录开始/结束时间、OBS 统计、插件状态和日志；不得发生崩溃、死锁、持续内存增长、连续推理失败或不可恢复的黑屏。
4. 在测试期间观察日志：不应出现连续的 GPU 读回、推理线程或资源泄漏错误；发生单次推理错误时，画面应自动进入兼容路径。

## 记录格式

每个平台至少保存以下信息：OBS 版本、系统版本、芯片/显卡、输入分辨率与帧率、质量模式、结果、统计截图或数值、日志文件路径和已知问题。使用 [验收记录模板](ACCEPTANCE_EVIDENCE_TEMPLATE.md)；不要在日志或提交中包含摄像头画面、个人身份信息或私有流地址。

## 平台覆盖

| 平台 | 最低验收 | 发布前额外要求 |
| --- | --- | --- |
| Apple Silicon macOS | macOS 12+、OBS 31+、摄像头或本地视频源 | Developer ID 签名与公证后的安装复测 |
| Windows x64 | Windows 10 22H2+、OBS 31+、摄像头或本地视频源 | 原生 x64 构建、安装与场景切换复测 |
